# Answer Modes Rubric

## `stable_concept`

Use for durable concepts that do not depend on the current patch. The answer should define the concept, explain practical meaning, and mark exact current values as out of scope.

## `verified_current_fact`

Use for exact current roster or move-specific facts grounded in `data/exports/`, `data/roster/`, or derived frame-current runtime assets.

## `strategy_or_matchup_knowledge`

Use for reviewed strategic or matchup claims. The answer should state scope, confidence, and patch sensitivity.

## `observation`

Use for video or replay observations. The answer should separate observed events from generalized claims.

## `unresolved_or_hold`

Use when evidence is missing, conflicting, patch-sensitive without current confirmation, or only present in review/third-party surfaces.
