# Blocking

## 要約

Blocking は、相手から離れる方向を入力して攻撃を guard する防御行動。SuperCombo Defense page は、standing / crouching / cross-up の guard direction、true blockstring 中の auto guard、proximity guard の性質を community source fact として説明している。

## Source claims

| Source | Claim | Confidence |
|---|---|---|
| [[sources/supercombo-street-fighter-6-defense]] | Blocking は相手から離れる方向、または下がり方向を hold して行う。Airborne 中や自分の attack recovery 中は block できない。 | medium |
| [[sources/supercombo-street-fighter-6-defense]] | Most attacks は standing / crouching のどちらでも block できるが、Overhead / High は standing block、Low は crouching block が必要。 | medium |
| [[sources/supercombo-street-fighter-6-defense]] | Cross-up は相手の後ろ側に当たる攻撃で、通常は forward guard を要求する。完全に cornered した相手には cross-up が成立せず、どちら方向でも block されると説明する。 | medium |
| [[sources/supercombo-street-fighter-6-defense]] | True blockstring 中は mid / overhead を auto guard するが、Low は low block が必要で、crouch block 中の overhead は auto-guard されない。 | medium |
| [[sources/supercombo-street-fighter-6-defense]] | Proximity Guard は攻撃 startup 中に back hold すると hit 前に block animation へ入る現象。SF6 では最大射程外からは通常発生しにくいと説明される。 | medium |

## 意味差分

- [[concepts/terms/blockstring]] は guard させる攻めの sequence。この page は defender 側の guard / block behavior を扱う。
- [[concepts/terms/drive-parry]] は high/low と left/right を自動的に受けられる Drive System action で、通常の blocking とは resource / reward が違う。
- Proximity Guard の range や例外は attack / projectile / hurtbox / spacing に依存するため、一般 rule として単純化しない。
