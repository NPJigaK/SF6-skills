# Street Fighter 6

## 要約

Street Fighter 6 は、この wiki が扱う glossary source、controls source、HUD source、Gauges source、Offense source、Defense source、Patch Notes source、公式 eSports BASE 用語解説、公式 Fighting Ground Battle System source、公式 frame-data source、公式 Battle Change List、SuperCombo community frame-data source の game context。現在、53 個別 term pages、Capcom 公式 eSports BASE scoped capture、Capcom 公式 Battle System scoped capture、SuperCombo controls / HUD / Gauges / Offense / Defense / Patch Notes web-page capture、SuperCombo Gauges numeric output、SuperCombo Patch Notes 17 version detail pages と community change-event index、Capcom 公式 Classic / Modern frame-data outputs 30 character data slugs 分、Capcom 公式 Battle Change List 20 version 分と公式 change-event index、SuperCombo 30 キャラ分の community frame-data source がある。公式 Classic + SuperCombo 補助データも 30 キャラ分作成済み。frame-data 取得物の配置と freshness は [[syntheses/frame-data-raw-layout]] と各 source page にまとめ、Gauges の数値は [[sources/supercombo-street-fighter-6-gauges]] と `wiki/outputs/data/gauges/supercombo/numeric-tables.json` で確認する。Offense の throw / safe jump / blockstring 系の値は [[sources/supercombo-street-fighter-6-offense]]、Defense の throw escape / wake-up / reversal / armor 系の値は [[sources/supercombo-street-fighter-6-defense]] で確認する。

SuperCombo Game Data web-page capture は、damage scaling、system data、range comparisons、Longest Punish `≤4F` などの tabbed table を保持する。Table text は `raw/web-pages/wiki.supercombo.gg/game-data/derived/tabber-tables.json` に source-preserving derived output として保存し、`≤5F` から `≤12F` の Longest Punish tabs は source 上で `(to-do)` として記録する。

## 関連する主張

- SuperCombo glossary は Drive System、frame data、juggle、notation の community terms を説明している。
- SuperCombo controls は Classic / Modern control type、numpad notation、classic notation、button nickname の community source として保持する。Modern Controls の damage penalty claim は公式確認前なので source fact として扱う。
- SuperCombo HUD は Battle HUD の表示要素、HUD icon、Health Bar Reference の community / visual source として保持する。navigation / character icon は HUD source content として扱わず、raw media download から除外した。HUD Icons section の 12 項目は [[concepts/terms/index]] 配下の term pages に display cue claim として集約する。
- SuperCombo Gauges は Drive / Super Gauge の cost、regeneration、cooldown、Drive Impact / Drive Rush distance table などの community numeric source として保持する。公式 Capcom source と重なる mechanics claim では公式 source を優先し、Gauges の数値は [[sources/supercombo-street-fighter-6-gauges]] と `wiki/outputs/data/gauges/supercombo/numeric-tables.json` に戻す。
- SuperCombo Offense は [[concepts/offense]]、[[concepts/terms/throw]]、[[concepts/terms/throw-loop]]、[[concepts/terms/safe-jump]]、[[concepts/terms/empty-jump-mixup]]、[[concepts/terms/blockstring]] に、通常投げ、throw invincibility、throw range、safe jump、empty jump、blockstring の community source claim を提供する。
- SuperCombo Defense は [[concepts/defense]]、[[concepts/terms/blocking]]、[[concepts/terms/throw-escape]]、[[concepts/terms/wake-up]]、[[concepts/terms/reversal]]、[[concepts/terms/armor]]、[[concepts/terms/anti-air]] に、blocking、throw escape、wake-up、reversal buffer、armor、anti-air、punish route の community source claim を提供する。
- SuperCombo Game Data は [[concepts/frame-data]] と [[concepts/terms/damage-scaling]] に、damage scaling progression、Super Art minimum、character-specific scaling tabs、system data、range comparison、Longest Punish `≤4F` table の community numeric source claim を提供する。
- Capcom 公式 eSports BASE scoped capture は、CPT / SFL 観戦向けの `実況ワード解説` として、[[concepts/terms/chip-damage|削り]]、[[concepts/terms/stun|スタン]]、[[concepts/terms/anti-air|対空]]、[[concepts/terms/corner|画面端]]、[[concepts/terms/cross-up|めくり]]、[[concepts/terms/raw-drive-rush|生ラッシュ]]、[[concepts/terms/punish-counter|パニッシュカウンター]]、[[concepts/terms/frame-advantage|有利フレーム]]、[[concepts/terms/hit-confirm|ヒット確認]]など 23 語を保持する。
- Capcom 公式 Fighting Ground Battle System scoped capture は Drive Gauge、Drive Impact、Drive Parry、Overdrive、Drive Rush、Drive Reversal、ゲーム画面 UI 説明を公式 source として保持する。
- [[concepts/terms/index]] は eSports BASE 由来 23 語に加えて、Burnout、Drive Parry、Drive Reversal、Drive Impact、Drive Rush Cancel、Overdrive、Super Art、Critical Art、damage scaling、wall bounce、air reset、chain、cancel、SuperCombo HUD icon 由来の display cue terms を個別 term page として保持する。
- Capcom 公式 frame-data captures は Classic / Modern の技データを raw snapshot と派生 JSON output として保持している。
- Capcom 公式 Battle Change List capture は 2023-07-24 update から 2026-05-28 update までの 20 version を raw HTML / Next.js data JSON として保持し、policy / common / fighter change rows に派生抽出している。
- SuperCombo Patch Notes capture は 2023-06-02 launch version から 2026-05-28 `2.0301` までの 17 version detail pages を raw wikitext / rendered DOM として保持する。16 rows は Capcom 公式 Battle Change id へ明示 link を持ち、`1.00` は launch version として official battle_change id を持たない。detail 本文は community `change-events.json` 1374 events / `move-change-index.json` 1118 targets に派生済み。
- SuperCombo 30キャラ分の frame-data 取得データは community source であり、公式にない range、juggle、notes、hitbox image refs などを後で統合する候補 data を含む。公式 Classic rows を正とする補助列付き output は 30 キャラ分ある。ただし複数候補、SuperCombo row 再利用、基本 field conflict、比較不能 field、条件付き field を持つ行には人間レビュー待ちの補助行が残り、`enrichment_review_queues` で理由を分離している。
- SuperCombo frame-data raw の source freshness / 取得時刻 / source revision は path ではなく manifest の `source_updated_at` / `captured_at_utc` / `source_revision` で分けて確認する。Capcom official raw の取得時点は manifest の `capture_label` / `created_at_utc` を参照する。
- 30 character data slugs の coverage は単一日付 snapshot ではない。JP は 2026-05-26、Ryu / Chun-Li / Zangief は 2026-05-27、残り 26 characters は 2026-05-30 の capture。
