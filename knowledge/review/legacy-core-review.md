# Legacy Core Migration Holding Review

> Temporary holding warning: this file is non-canonical review material. Do not use entries here as final answer evidence, do not generate runtime knowledge from it, and do not treat it as accepted v2 taxonomy.

## Held Material

The stable concept portions of the legacy note were migrated into curated concept and glossary pages. The items below are held because they become ambiguous, exact-current, patch-sensitive, insufficiently supported, or conflicting unless they are reviewed as separate claims.

## Move-Specific Timing Claims

- Any exact startup, active, recovery, total duration, hit advantage, or block advantage value for a named move.
- Any statement that a specific move is punishable by a specific response in all current situations.
- Any statement that compares exact timing values between specific character options.

Review requirement: move-specific timing belongs with current-fact exports or separately reviewed current claims, not curated concept pages.

## Character-Specific Hit Confirm Claims

- Exact confirm windows.
- Optimal routes after a confirm.
- Claims that a particular sequence is safe to attempt on block.
- Claims that a particular visual cue is consistently reactable for all players.

Review requirement: separate player-execution guidance from current move behavior and route evidence.

## Wake-Up Pressure And Setup Claims

- Exact meaty setup timing.
- Guaranteed strike, throw, or bait options after a named knockdown.
- Claims about which defensive options lose to a setup in the current patch.
- Character-specific wake-up pressure routes.

Review requirement: record knockdown source, spacing, timing method, defender options, and current move evidence before promotion.

## Community Term Application Claims

- Claims that a shimmie is official game terminology.
- Claims that a specific shimmie route is guaranteed in a named matchup or current setup.
- Claims that a throw-bait situation is universal across characters or spacing.

Review requirement: keep the glossary definition separate from setup-specific strategy claims.

## Deprecated Legacy Metadata

Legacy source rankings, bracket answer labels, and old claim buckets were discarded during migration. Do not promote those labels into v2 metadata. Use the generic v2 fields shown in this file and in curated pages.
