# Triage new SF6 notes

Use `$sync-knowledge`.

Read any newly added notes, summaries, links, or pasted research in this repo.
Classify each candidate item as one of:
- core
- mixed
- current-fact
- community-term
- insufficient-evidence

Update only:
- `skills/kb-sf6-core/references/KNOWLEDGE.md`
- `skills/kb-sf6-core/references/REVIEW_QUEUE.md`

Rules:
- Promote only stable core concepts into `KNOWLEDGE.md`.
- Send mixed/current-fact/weak-evidence items to `REVIEW_QUEUE.md`.
- Do not add exact numbers or patch-sensitive facts to `KNOWLEDGE.md`.
- If an item duplicates an existing concept, merge instead of creating a new section.
- Summarize what changed and why.
