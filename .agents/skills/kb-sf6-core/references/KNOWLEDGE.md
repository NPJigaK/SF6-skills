# SF6 Core Knowledge

このファイルは **不変の概念** だけを保持する。
current fact / exact 数値 / パッチ差分は含めない。

---

## フレーム
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: ゲーム内の時間を区切る最小単位。
- 実戦的意味: 技の速さ、先に動けるか、反撃が間に合うかを考える共通の物差し。
- 境界: 特定技の「何Fか」や current の仕様差は別確認。

## 発生・持続・硬直
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: 発生は当たり始めるまで、持続は当たり判定が出ている間、硬直はその後に動けない時間。
- 実戦的意味: 発生は速さ、持続は置きや重ね、硬直はスカりやガード後の危険さに関わる。
- 境界: 特定技の具体値は別確認。

## 全体Fと硬直差
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: 全体Fは技全体の長さ、硬直差はヒット/ガード後にどちらが先に動けるかの比較結果。
- 実戦的意味: 全体Fが長いから不利、短いから有利、とは限らない。
- 境界: 個別技の exact 計算は別確認。

## ヒット硬直とガード硬直
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: ヒット硬直は相手が被弾後に動けない時間、ガード硬直は防御後に動けない時間。
- 実戦的意味: 有利不利、連携継続、コンボ成立に関わる。
- 境界: current の具体数値は別確認。

## 有利・不利
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: `+` は自分が先に動ける、`-` は相手が先に動ける。
- 実戦的意味: 連携継続、暴れ潰し、確反の判断基準。
- 境界: 特定技の有利不利は別確認。

## 確反
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: 確定反撃。相手の不利が大きく、こちらの攻撃が必ず間に合う状況。
- 実戦的意味: 相手の不利Fと自分の発生を照合して判断する。
- 境界: どの技が確反になるかは相手技・距離・条件次第で別検証。

## ヒット確認
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: 最初の技がヒットしたのを見てから次を出し分けること。
- 実戦的意味: 当たったときだけ伸ばし、ガードされたら止める判断。
- 境界: キャラ別の確認猶予や最適ルートは別確認。

## 起き攻め
- 種別: core
- 根拠階層: concept-stable
- 回答ラベル: `[概念のみ]`
- 一言定義: 相手の起き上がりに攻めを重ねて択を迫ること。
- 実戦的意味: 打撃、投げ、様子見などで防御択に圧をかける。
- 境界: 何が重なるか、どの択が成立するかは状況依存で別確認。

## シミー
- 種別: core-community-label
- 根拠階層: community-labeled
- 回答ラベル: `[概念のみ]`
- 一言定義: 投げに行くように見せて少し下がり、相手の投げ抜けや遅らせ投げ抜けを空振らせてから差し返す動き。
- 実戦的意味: 相手の投げ警戒を逆手に取る。
- 注意: 公式用語ではなく、コミュニティで一般的な言い方。
- 境界: どの状況で通るかは別検証。
